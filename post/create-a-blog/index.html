一.首先肯定是要先下载go语言安装包 🔗

  • 1.找到linux 版本go包
wget https://golang.google.cn/dl/go1.21.4.linux-amd64.tar.gz ```

```tar -C /usr/local -zxvf  go1.17.5.linux-amd64.tar.gz```

   > 习惯用vim，没有的话可以用命令sudo apt-get install vim安装一个

##  二.安装Hugo
- 1.首先下载
   下载地址：wget https://github.com/gohugoio/hugo/releases/download/v0.80.0/hugo_0.80.0_Linux-64bit.deb
（这是终端的命令）

- 2.安装Hugo

    终端：sudo dpkg -i hugo_0.80.0_Linux-64bit.deb
    
- 3.检查是否安装成功	

    终端：hugo version

## 三.在本地搭建静态网站

- 1.首先用hugo新建一个site
 
    终端：hugo new site myblog（myblog是自定义的名字）
    
- 2.下载主题
    主题官网：https://themes.gohugo.io ，找到想要的主题，点击去，复制下载命令，下载到 myblog 目录下的 themes 目录下
你也可以压缩包下载后直接复制粘贴到theme文件夹里面。（注意这个时候要把myblog里面的config.toml在最后面添加theme = “你的主题文件名字”，并把第一行里面的链接删除）

- 3.在本地启动博客
   在 myblog 目录下命令：hugo server
   
   之后会出现这样：Web Server is available at http://localhost:1313/ (bind address 127.0.0.1)
   
   将localhost:1313放到本地浏览器中应该就可以看见自己做的网页了。
   
## 四.写一篇文章并上传到github上
- 1、生成md文件

     在 myblog 目录下
      终端：hugo new post/blog.md（blog.md可以是自己定义名字）
     
     打开blog.md，将其中的true改成false让这个文本能被看到
     
- 2.新建一个github仓库

     ①新建一个仓库

     ②填写仓库地址，填写格式为：用户名小写.github.io ，例如我的仓库地址：wujvya.github.io ，那么我就可以通过 https://wujvya.github.io 来访问我的博客

     ③创建仓库
     
- 3.生成静态网页
    
    hugo  命令可以将你写的 Markdown 文件生成静态 HTML 网页，生成的 HTML 文件默认存放在 public 文件夹中。
    终端：hugo
    
    因为hugo生成的静态 HTML 网页文件默认存放在public文件中，
    
    所以推送网页内容只需要把public中的HTML网页文件发布到GitHub Pages 仓库中。

    将public文件夹初始化为 Git 仓库，并设置默认主分支名为  main。这么做的原因是：

    GitHub 创建仓库时生成的默认主分支名是 main
    用 git init 初始化 Git 仓库时创建的默认主分支名是 master
    将 git init 创建的 master 修改成 main ，再推送给远端仓库 <username>.github.io ，这样才不会报错。

    终端：cd public
    git init -b main

    
- 4.关联ssh
   将 public  文件夹关联远程 GitHub Pages 仓库，使用 GitHub Pages 仓库的 SSH 链接。

   （ ⚠️ 注意：要让 SSH 链接起作用，需要你添加过 SSH Key。如果你没有设置过 SSH Key，请参考 https://blog.csdn.net/muxuen/article/details/124662978?ops_request_misc=&request_id=&biz_id=102&utm_term=linux%E9%85%8D%E7%BD%AEssh%20github&utm_medium=distribute.pc_search_result.none-task-blog-2~all~sobaiduweb~default-3-124662978.142^v99^pc_search_result_base8&spm=1018.2226.3001.4187

   千万记住ssh一定要打开 可以参考：1.https://blog.csdn.net/m0_67402970/article/details/126434165


    2. https://blog.csdn.net/shenxinde/article/details/124119911
    终端输入：git remote add origin (你仓库导航栏地址+.git)
    
- 5.推送博客源仓库的  public  文件夹中的 HTML 网页文件到 GitHub Pages 仓库 中，在推送仓库内容前要先用
    git pull --rebase origin main 和远端仓库同步，否则会报错。
    
    git pull --rebase origin main

    git add .

    git commit -m "...(修改的信息)"
    
    git push origin main
    
    
 ### ps：用push有可能不能推送，会显示：Please use a personal access token instead.
